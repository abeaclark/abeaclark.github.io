Arrays and Hashes

Hash or Array?

Hashes and arrays are important data structures in programming. But, what exactly is the difference between a hash and an array? Both structures are used to organize data for easy recall and manipulation. The structure of each makes it especially helpful for certain applications. Here is a quick overview of the differences:

Arrays
  • List is ordered and each data point is assigned an index number
  • Index's start at 0 and correspond to the order that items were added to the array (Note that there are methods by which you can add items at the beginning, end, or middle of an array)
  • You can call a given value from the array by specifying its index number like this: array[7]
  • Great for storing data where order matters (ex: stock prices over time) or where the exact meaning of any individual data point is not extremely important to the whole (ex: recording temperature each second in order to display a graph of the temperature change in a given day -- each individual second will not be examined individually)
Hashes
  • List is unordered and each data point is assigned a corresponding key (think of it as a name to remember the data point by, like "password" or "username")
  • Hashes are organized into pairs that looke like this: hash = [key => data]
  • You can call on a piece of data in the hash like this hash[key] which will return "data"
  • Great for storing data that has individual significance such as usernames and passwords but that isn't sequentially important
  • This examples shows possible uses for an array and a hash. The array is used to represent a next-7-day forecast and the hash stores relevant contact information
  • It would not make sense to add a key like "Thursday" or "May24" to the temperature_forecast array because you know that it corresponds to the next 7 days and you have more flexibility if you know the forecast relative to today
  • The hash makes more sense for the contact information because instead of having to remember that "Website" is the second element, you can just use the :website key
