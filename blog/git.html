The terminal

The terminal is a different (actually, the original) way of accessing/running files on your machine and a powerful tool for searching, manipulating, and syncing data. The interface and features are similar to the first computers ever created. Interaction with files does not happen visually (think, mouse and screen), but rather by typing commands and reading the corresponding outputs.

The terminal interface is non-intuitive to most--if not all--beginners. When I first started working with it, I was confused why anyone would choose it over the beautiful visual interfaces that Apple and Microsoft have created and [perfected]. As you will find out by working with the terminal, it does offer many synergies that can improve your productivity and give you a much deeper understanding of how a computer works. The following resources are great starting places for learning the basics of the terminal (Always open to suggestions to add to this list!).

git and GitHub

Git is a version control method used by programming teams and individuals. It allows multiple people to collaborate on the same code base while minimizing the chance of duplicated work or lost work due to versions being saved over. Git has empowered the open-source software movement by allowing programmers across the globe to contribute effeciently to the same project.

If you have used dropbox, google docs, or other similar software, you are familiar with some of the concepts in git. Here are the main points:

  • There is one "Master" version of a document at any given time--this is the most perfect version that is production ready
  • New additions to the software or fixes are done on "branches" of the Master version
  • A branch is a duplicate of the master version that an individual programmer works from, making changes and additions
  • When finished making changes, a programmer uploads the branch and it is programmatically compared to the Master version
  • Any changes are incorporated into the current Master version
  • If there are conflicts (ex: if someone else has made a conflicting edit in the meantime) then a flag is raised and the users can decide which edit to keep
  • Uploading changed branches is a two step process so that more than one programmer looks at the code before it is published to the Master (ex: Coder A uploads the changes, Coder B approves the changes and merges them into the Master)

GitHub is a cloud-based host for code repositories that follows the git protocol. Instead of hosting the code on a local server, teams can collaborate via the cloud remotely. GitHub can interface with the command line via the prefix "git" and also has a visual online interface. For programmers, the command line interface is vastly superior.
