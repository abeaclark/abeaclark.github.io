Ruby Classes

So far, we have three major data structures:

  1. Arrays:

    Useful for ordered data that does not need to be closely associated to a pair (ex: stock prices, temperature over time, names on a waiting list queue, etc.)

  2. Hashes:

    Useful for key:value pairs that need to be associated (ex: usernames and passwords, label and value, etc.)

  3. Classes:

    Useful for assigning multiple values and specialized methods to a data structure (Ex: users will have name, username, password, statistics about usage, metadata, and will need specialized methods such as printing profile information, deleting posts, etc.)

Let's look deeper into classes. Classes are extremely versatile and widely used. You've already been using classes that are built into the Ruby language such as "string" "integer" "object" and others. Did you know these were classes?

Remember how there are some methods you can call on integers that you can't call on strings? (ex: #even? , #round(), etc.). This is because the makers of the Ruby language have defined certain methods that can only be used within the "Integer" class. They have set the "Integer" class apart from other classes because there are certain methods that only make sense to call on this class (you wouldn't want to call the #round method on a string, it wouldn't make any sense!)

Classes are hierarchal (like a family tree). The highest level of class is object. Classes inherit methods and other characteristics from parent classes. A class above the current class is called a "superclass". See the example below that clarifies the hierarchal structure.

Classes often have attributes that need to be assigned, retrieved, or overwritten. Instead of needing to write separate methods each time you want to read and/or write a value to a class, Ruby has this functionality built into the attr_reader, attr_writer, and attr_accessor tags. The reader and writer tags give you access to read-only and write-only priveledges, and the accessor tag allows you to read/write.
